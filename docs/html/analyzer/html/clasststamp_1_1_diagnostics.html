◆ fTime0

uint32_t tstamp::Diagnostics::fTime0

Initial event time (begin of run)

Information in this class is updated every time an event is inserted into the queue. Typically, one passes an instance of this class to a queue which will handle the updating of diagnostic inforamtion automatically whenever a new event is pushed into the queue.

Note
When tstamp::Queue handles the updating of this class's fields, the updates happen at the end of a Push(). This means that anything resulting from a push, be it a new coincidence match or singles event processed will be reflected in the state of the Diagnostics instance. Information is also updated when flushing from the queue, but here tims_diff is set to zero since no new events are incoming.
