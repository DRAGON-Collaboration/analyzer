Unpack a generic V1190 buffer.

Parameters
[in]pbufferPointer to the buffer data
[in]bankNameName of the midas bank being unpacked
Returns
True if the buffer was successfully unpacked, false otherwise

V1190 buffers are 32 bit words. Bits 27-31 specify the type of data contained in the buffer. In this function, we read the buffer type and then handle appropriately.

case GLOBAL_HEADER: read event counter from bits 5 - 26

case GLOBAL_TRAILER: read status word from bits 24-26 and word count from bits 5-21

case EXTENDED_TRIGGER_TIME: read extended trigger from bits 0 - 26

case TDC_HEADER: read bunch id from bits 0 - 12, event id from bits 12 - 23

case TDC_MEASUREMENT: See unpack_data_buffer()

case TDC_ERROR: See handle_error_buffer()

case TDC_TRAILER: See unpack_footer_buffer()

Bail out if we read an unknown buffer code

bool vme::V1190::unpack_data_buffer ( const uint32_t *const  pbuffer)
private

Unpack event data buffer.

Parameters
[in]pbufferPointer to the data buffer word.

A data buffer encodes the measurement value (i.e. the pulse time) for a single TDC measurement. The TDC is multi-hit, so more than one measurement per channel can be read in a single event.

See below for bitpacking instructions.

  • Bit 26 tells the measurement type (leading or trailing)
  • Bits 19-25 tell the channel number
  • Bits 0 - 18 encode the measurement value
